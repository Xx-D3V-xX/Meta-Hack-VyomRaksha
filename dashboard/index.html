Phase 1 — Training Loss (Threat Agent, Qwen2.5-14B)
93% loss reduction in 300 GRPO steps. SFT warmup (25 demos × 3 epochs) followed by GRPO with IsolatedResourceEnv verifiable reward. Hardware: AWS g5.2xlarge, NVIDIA A10G 24GB, ~21s/step.
Phase 1 — Token Accuracy (Threat Agent)
Token accuracy: 47.9% → 94.7% (+97.7%). The agent learned to produce well-formed JSON recommendations with correct action selection, urgency calibration, and structured reasoning.
Phase 1.5 — Reward Signal (Threat Agent, Joint Exposure)
Phase 1.5 exposes agents to cross-domain scenarios. Reward variance (std 0.15–0.29) reflects active learning across diverse coordination scenarios — higher variance than Phase 2 is expected and healthy.
Phase 2 — SarvaDrishti Orchestrator Reward (Original Run, 400 steps)
Reward 0.921–0.947 across 400 steps. Low variance (std ~0.003) indicates a converged coordination policy — SarvaDrishti consistently produces high-quality arbitration decisions. This is the expected result for a well-trained orchestrator.
Reward Model Training (Bradley-Terry, Qwen2.5-3B, 150 steps)
Preference reward model trained on 164 SarvaDrishti coordination preference pairs. Accuracy reached 87.5–100% per batch. Margin (preferred vs rejected score gap) increased from 0.16 to 0.74 at peak. Used to score SarvaDrishti deliberation quality in Phase 2.

Reward Design — 3-Layer Architecture
LayerSignalRange
Outcome (dominant)Probe survival ±10, objectives ±8, threat ±3/−5−15 to +20
Shaped (capped)Emergency accuracy, conflict quality, urgency calibration0 to 0.90/ep
Learned (SarvaDrishti)Bradley-Terry preference model (Qwen2.5-3B, 164 pairs)0 to 1.0
Anti-gaming constraint: Shaped rewards always sum to less than the smallest outcome reward. Passive agents score <0.15 on Tasks 4–5 by design.

System Architecture
VyomRakshaEnvironment (OpenEnv-compliant FastAPI)
├── R2ProbeSimulator          — 7-resource physics-lite simulation
│   ├── power, fuel, thermal, compute_budget
│   ├── structural_integrity, radiation_integrity
│   └── data_buffer, comms_bandwidth, instrument_health
│
├── MultiAgentLoop            — 12-step coordination cycle per environment step
│   ├── 8× SubAgent           — domain-specialist policies (GRPO-trained QLoRA)
│   │   ├── Observe own domain only (domain isolation)
│   │   ├── Recommend action with urgency + confidence
│   │   └── Check emergency authority threshold
│   │
│   ├── EmergencyHandler      — pre-deliberation scan + priority ordering
│   ├── ShadowSimulator       — counterfactual eval for emergency rewards
│   │
│   └── SarvaDrishti          — orchestrator (GRPO-trained Qwen2.5-14B)
│       ├── ConflictResolver  — 5 conflict types, irreversibility ranking
│       └── StrategyManager   — 5 strategies, reactive + proactive updates
│
└── R2RewardCalculator        — 3-layer reward (outcome + shaped + learned)
